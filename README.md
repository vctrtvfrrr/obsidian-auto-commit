# Obsidian - Auto Commit

Obsidian plugin that automatically commits and pushes vault changes to Git after a configurable period of inactivity. Commit messages are generated by Claude AI based on the staged diff.

Desktop only. Requires Git in PATH.

## How it works

1. Any file change in the vault (create, modify, delete, rename) resets an inactivity timer.
2. When the timer expires with no further changes, the plugin runs `git add -A`, generates a commit message via the Anthropic API, and commits.
3. If push is enabled, it fetches the remote, rebases if needed, and pushes.

On startup, if there are uncommitted changes from a previous session, a commit is triggered immediately.

## Installation

### Prerequisites

- Git installed and available in PATH
- An [Anthropic API key](https://console.anthropic.com/)
- Node.js and npm (for install script only)
- `~/.npmrc` authenticated against the Gitea registry (read token):
  ```
  @vctrtvfrrr:registry=https://git.codelab.tec.br/api/packages/vctrtvfrrr/npm/
  //git.codelab.tec.br/api/packages/vctrtvfrrr/npm/:_authToken=<your-read-token>
  ```

### Install script

```bash
# Install into ~/obsidian (default)
./install.sh

# Install into a different vault
./install.sh /path/to/your/vault
```

Then in Obsidian: **Settings → Community plugins → Enable Auto Commit**.

To update, run `./install.sh` again after a new release.

## Configuration

Open **Settings → Auto Commit**:

| Setting                       | Default            | Description                                              |
| ----------------------------- | ------------------ | -------------------------------------------------------- |
| Inactivity interval (minutes) | `15`               | How long to wait after the last change before committing |
| Branch                        | _(current branch)_ | Branch to push to; leave empty to use the current branch |
| Remote                        | `origin`           | Git remote name                                          |
| Push after commit             | on                 | Automatically push after each commit                     |
| Anthropic API key             | —                  | API key used to generate commit messages                 |

Settings are stored obfuscated in `.obsidian/plugins/obsidian-auto-commit/data.json` to avoid leaking the API key through Git diffs.

### Manual trigger

The command **Auto Commit: Run now** (accessible via the command palette) triggers a commit immediately, respecting all the same guards as the automatic timer.

## Edge cases

- **Special repo state** (merge, rebase, cherry-pick, bisect, detached HEAD): skipped silently.
- **No changes**: no-op.
- **Diff > 50 KB**: aborted with a persistent notice; no commit is created.
- **AI unavailable**: aborted with a persistent notice; staged changes remain for the next cycle or manual commit.
- **Rebase conflict on push**: rebase is aborted, local commit is preserved, persistent notice shown.
- **Push failure**: local commit is kept; next cycle will retry the push.
- **Concurrent runs**: if a commit is already in progress, the new trigger is ignored.

## Development

### Setup

```bash
npm install
```

### Build

```bash
npm run build      # production build → main.js
npm run dev        # watch mode
npm run typecheck  # type-check only, no output
```

### Testing locally

Copy the plugin files to your vault after each build:

```bash
VAULT=~/obsidian
PLUGIN_DIR="${VAULT}/.obsidian/plugins/obsidian-auto-commit"
mkdir -p "${PLUGIN_DIR}"
cp manifest.json main.js "${PLUGIN_DIR}/"
```

Then reload the plugin in Obsidian (**Settings → Community plugins → disable → enable**).

### Release

1. Bump the version:
   ```bash
   npm version patch   # or minor / major
   ```
2. Push the tag:
   ```bash
   git push --follow-tags
   ```

The Gitea Actions workflow (`.gitea/workflows/release.yml`) picks up the `v*` tag, builds, and publishes to the npm registry automatically. Requires the `NODE_AUTH_TOKEN` secret (write token) configured in the repo settings.
